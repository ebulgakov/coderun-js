Управление конкурентностью (Concurrency)

Описание:

-

Требования:

  1. На вход передается массив функций, которые возвращают промисы (чтобы они не начали выполняться сразу при создании).
  2. Как только один промис завершается, на его место тут же должен запускаться следующий из очереди (а не ждать, пока завершится весь "батч" из limit штук).
  3. Результат должен быть массивом ответов в исходном порядке.
  4. Если хоть один упал — падаем все.
